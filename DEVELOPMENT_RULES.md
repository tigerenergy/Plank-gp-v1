# 🚨 개발 규칙 (반드시 준수)

## 1. 상태 관리

- ❌ `useState`, `useEffect` 남발 금지
- ✅ **Zustand**로 상태 관리 중앙화
- ✅ React Compiler 사용 중이므로 `useCallback`, `useMemo` 불필요

## 2. 카드 생성 규칙

- ✅ **카드 만든 사람 = 자동으로 담당자(assignee)**
- 담당자 변경은 나중에 관리자만 가능하게 구현 예정

## 3. 댓글 규칙

- ✅ **누구나 댓글 작성 가능**
- ✅ **모든 댓글이 모두에게 보임**
- ✅ 본인 댓글만 수정/삭제 가능

## 4. 체크리스트 규칙

- ✅ 체크리스트 추가 시 **기존 값 유지**
- ❌ 상태 업데이트로 기존 데이터 삭제 금지
- ✅ 새 항목은 기존 배열에 **추가(append)**

## 5. 협업 서비스 원칙 (Trello 방식)

- ✅ 로그인한 모든 사용자 = 팀원
- ✅ 모든 팀원이 모든 보드 접근 가능
- ✅ 서로 댓글 달고 체크리스트 공유
- ✅ 담당자 할당으로 협업

## 6. 애니메이션

- ✅ **framer-motion** 사용
- ❌ CSS 애니메이션 직접 작성 금지

## 7. 스타일링

- ✅ Tailwind 관련 설정은 `tailwind.config.ts`에서 관리
- ❌ `globals.css`에 Tailwind 관련 설정 넣지 않기
