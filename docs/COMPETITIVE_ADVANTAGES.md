# 🚀 Plank의 경쟁 우위 및 차별화 포인트

> **Plank**는 Trello와 유사하지만, **관리자 없는 독자적인 협업 플랫폼**을 지향합니다.

---

## 🎯 핵심 차별화 포인트

### 1. 완전한 투명성: 상급자와 직원이 서로 볼 수 밖에 없는 구조 ⭐ **최대 차별화 포인트**

**핵심 가치**: **상급자들도 놀 수가 없음** - 이것이 Plank의 핵심 포인트입니다.

**Trello의 한계:**
- 보드 소유자/관리자가 모든 권한을 가짐
- 관리자가 멤버의 작업을 수정/삭제할 수 있음
- 계층적 권한 구조로 인한 불평등
- 관리자는 직원의 작업만 볼 수 있지만, 자신의 작업은 숨길 수 있음
- 직원은 관리자의 작업을 제대로 볼 수 없어 불평등감 발생

**Plank의 해결책:**
- ✅ **상급자(관리자)와 직원이 서로 볼 수 밖에 없는 구조**
  - 상급자의 작업도 직원들에게 완전히 공개됨
  - 직원의 작업도 상급자에게 완전히 공개됨
  - **상급자들도 놀 수 없음** - 자신의 작업이 직원들에게 보이므로 책임감이 생김
- ✅ **내가 만든 건 내가 수정만 가능** (작성자만 수정/삭제 가능)
- ✅ **누구나 볼 수 있지만, 내 작업은 내가 통제**
- ✅ 관리자가 임의로 수정하거나 삭제할 수 없음
- ✅ **양방향 투명성**: 상급자도 직원도 서로의 작업을 볼 수 있어 누구도 게으름을 피울 수 없음

**실제 사용 사례:**
- 주간보고에서 관리자가 성과를 지우고 보고하는 문제 해결
- 상급자의 작업도 직원들에게 보여서 "상급자도 열심히 일하고 있구나" 확인 가능
- 직원의 작업도 상급자에게 보여서 "직원도 열심히 일하고 있구나" 확인 가능
- **상급자도 놀 수 없음**: 자신의 작업이 직원들에게 보이므로 책임감이 생기고, 실제로 일을 해야 함
- 각자의 작업에 대한 소유권과 책임 명확화
- 투명하고 공정한 협업 환경 조성

---

### 2. 통합 주간보고 기능 ⭐ **Trello에 없는 기능**

**Trello의 한계:**
- 작업 관리만 가능, 보고 기능 없음
- 완료된 작업만 추적 가능
- 진행 중인 작업의 상태를 체계적으로 보고하기 어려움

**Plank의 해결책:**
- ✅ **주간보고 작성 기능** (완료/진행중 모두 포함)
- ✅ **웹페이지로 주간보고 공유** (메일 불필요)
- ✅ **개개인이 뭐뭐 했는지 다같이 볼 수 있는 페이지**
- ✅ **진행 중인 작업의 현재 상태, 진척도, 이슈사항 기록**
- ✅ **과거 주간보고 데이터 보관** (나중에 확인 가능)

**실제 사용 사례:**
- 엑셀 시트 + 관리자 복붙 방식의 문제점 해결
- 실시간으로 팀원들의 작업 현황 확인
- 대면 회의 전에 미리 파악 가능

---

### 3. 실시간 협업 + 투명성

**Trello의 한계:**
- 실시간 업데이트는 있지만, 투명성 부족
- 관리자가 특정 작업을 숨기거나 수정 가능

**Plank의 해결책:**
- ✅ **모든 멤버가 모든 작업 조회 가능**
- ✅ **작성자만 수정 가능** (투명성 + 소유권 보장)
- ✅ **실시간 알림 시스템**
- ✅ **완료된 작업 통계 및 추적**

## 💡 추가 개선 아이디어

### 1. 작업 히스토리 및 감사 로그 (Audit Log)

**아이디어:**
- 모든 작업 변경 이력을 자동으로 기록
- 누가, 언제, 무엇을 변경했는지 추적
- 삭제된 작업도 일정 기간 보관 (복구 가능)

**차별화 포인트:**
- Trello는 히스토리가 제한적
- Plank는 완전한 투명성과 책임 추적

**구현 방안:**
```sql
CREATE TABLE activity_logs (
  id UUID PRIMARY KEY,
  board_id UUID,
  user_id UUID,
  action_type VARCHAR(50), -- created, updated, deleted, moved
  entity_type VARCHAR(50), -- card, list, comment
  entity_id UUID,
  changes JSONB, -- 변경 전/후 내용
  created_at TIMESTAMPTZ
);
```

### 2. 개인 작업 대시보드

**아이디어:**
- 각 멤버가 자신의 작업만 필터링해서 볼 수 있는 대시보드
- 내가 담당한 카드, 내가 작성한 댓글, 내 주간보고 등
- 개인 생산성 추적 (선택사항)

**차별화 포인트:**
- Trello는 보드 중심, 개인 중심 뷰 부족
- Plank는 개인과 팀의 균형

---

### 3. 스마트 알림 시스템

**아이디어:**
- 중요도 기반 알림 (내가 담당한 카드 우선)
- 알림 그룹화 및 요약
- 알림 설정 커스터마이징 (이메일, 브라우저, 앱 푸시)

**차별화 포인트:**
- Trello는 알림이 단순함
- Plank는 스마트한 알림으로 정보 과부하 방지

---

### 4. 템플릿 및 워크플로우 자동화

**아이디어:**
- 보드 템플릿 (스프린트, 버그 트래킹, 프로젝트 관리 등)
- 자동화 규칙 (카드 이동 시 자동 알림, 완료 시 자동 보고 등)
- 워크플로우 시각화

**차별화 포인트:**
- Trello는 Power-Ups로 복잡함
- Plank는 간단하고 직관적인 자동화

---

### 5. 통합 및 API

**아이디어:**
- GitHub, Slack, Google Calendar 연동
- RESTful API 제공 (서드파티 통합)
- Webhook 지원 (이벤트 기반 통합)

**차별화 포인트:**
- Trello는 API가 있지만 복잡함
- Plank는 개발자 친화적인 API

---

### 6. 모바일 최적화

**아이디어:**
- PWA (Progressive Web App) 지원
- 모바일 네이티브 앱 (React Native)
- 오프라인 모드 지원

**차별화 포인트:**
- Trello 모바일 앱은 기능이 제한적
- Plank는 모바일에서도 완전한 기능 제공

---

### 7. 다국어 지원

**아이디어:**
- 한국어, 영어 기본 지원
- 사용자 선택 언어로 UI 변경
- 다국어 주간보고 템플릿

**차별화 포인트:**
- Trello는 한국어 지원이 부족
- Plank는 한국 시장에 특화

---

### 8. 데이터 내보내기 및 백업

**아이디어:**
- 모든 데이터 JSON/CSV 형식으로 내보내기
- 자동 백업 (일일/주간)
- 데이터 마이그레이션 도구 (Trello → Plank)

**차별화 포인트:**
- Trello는 데이터 내보내기가 제한적
- Plank는 완전한 데이터 소유권 보장

---

### 9. AI 기반 인사이트

**아이디어:**
- 작업 패턴 분석 (언제 가장 생산적인지)
- 예상 완료일 추정 (과거 데이터 기반)
- 리스크 감지 (마감일 지연 가능성 알림)
- 주간보고 자동 요약

**차별화 포인트:**
- Trello는 AI 기능 없음
- Plank는 AI로 더 스마트한 협업

---

### 10. 프라이버시 및 보안 강화

**아이디어:**
- 엔드투엔드 암호화 옵션
- 2FA (Two-Factor Authentication)
- IP 화이트리스트
- 감사 로그 (누가 언제 접근했는지)

**차별화 포인트:**
- Trello는 보안이 기본 수준
- Plank는 엔터프라이즈급 보안 옵션

## 💰 수익화 전략

> **상세 전략**: [`BUSINESS_STRATEGY.md`](./BUSINESS_STRATEGY.md) 참고

### 1. Freemium 모델
- **무료**: 기본 보드 3개, 멤버 5명까지, 기본 주간보고
- **Pro ($9/월 또는 ₩10,000/월)**: 무제한 보드, 멤버, 고급 주간보고 기능
- **Enterprise ($29/월 또는 ₩30,000/월)**: SSO, 고급 보안, 우선 지원

### 2. 주간보고 기능 (핵심 차별화 포인트)
- **무료**: 기본 주간보고 (자동 수집 + 보완 입력)
- **Pro**: AI 요약, 통계 분석, 내보내기, 템플릿, 알림
- **Enterprise**: 고급 통계, 커스텀 템플릿, 자동 이메일 전송

### 3. API 및 통합
- **무료**: 기본 API (제한적)
- **Pro**: 무제한 API, Webhook
- **Enterprise**: 전용 API, 커스텀 통합
